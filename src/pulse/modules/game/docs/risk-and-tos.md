# Risk And TOS

商业游戏通常禁止第三方脚本。启用真实点击表示用户理解并接受账号处罚风险。

## 1. 默认边界

| 项 | 规则 |
|---|---|
| 默认模式 | `dry_run=true` |
| 执行频率 | 每游戏默认一天一次 |
| 任务范围 | 签到、月卡、邮件、任务图、免费/半价抽卡 |
| 风控页面 | 命中滑块、验证码、账号异常模板直接 abort |
| 通知 | 失败与风险控制事件推 Notifier |

## 2. 不做范围

- 指纹修改
- 进程隐藏
- 验证码 / 滑块绕过
- 协议逆向
- 抓包改包
- 批量账号
- 多开对抗
- PVP / 战斗 / 部队调动
- 自动登录

## 3. 配额

半价抽卡以 `daily_max_pulls` 控制,语义是"每日最多点击 N 次半价抽卡按钮"。超过配额进入 SafetyPlane ASK,不自动继续。

## 4. 真实执行门槛

| 门槛 | 要求 |
|---|---|
| 模板 | 人工确认点击位置 |
| dry-run | 至少完整跑通一次 |
| 风险模板 | 滑块 / 验证码 / 账号异常模板已配置 |
| 配额 | 抽卡任务明确 `daily_max_pulls` |
